chris_plugin

Python ChRIS Plugin Support

chris_plugin is a library for writing ChRIS plugins.

Getting Started

Take a look at the README and examples.

A walk-through can be found on our wiki.

Core API

chris_plugin is a decorator which transforms an ordinary python function into a ChRIS plugin.

from chris_plugin import chris_plugin
from argparse import ArgumentParser

parser = ArgumentParser()
parser.add_argument('--name', required=True)

@chris_plugin(parser=parser, title='Example')
def main(options, inputdir, outputdir):
    print(f'hello, {parser.name}')

Helper Functions

This library also provides helper functions for common patterns, such as chris_plugin.PathMapper.

import shutil
from chris_plugin import chris_plugin, PathMapper

@chris_plugin
def main(_, input_dir, output_dir):
    for input_file, output_file in PathMapper.file_mapper(input_dir, output_dir):
        print(f'Copying {input_file} to {output_file}')
        shutil.copy(input_file, output_file)

Creates a decorator which identifies a ChRIS plugin main function and associates the ChRIS plugin with metadata.

When called, CLI arguments are parsed and passed to the decorated function as its first argument. It is also given the data directories as pathlib.Path, depending on the type of plugin:

  • "fs" plugins are given one data directory
  • "ds" and "ts" plugins are given two data directories

All data directories are made sure to exist, and the output directory is first created if needed.

 97class PathMapper(Iterable[Tuple[Path, Path]]):
 98    """
 99    An iterator which discovers input files in a directory and maps
100    them to output path names.
101
102    For example, an input file `/share/incoming/a/b/c.txt` will be
103    mapped to `/share/outgoing/a/b/c.txt`.
104
105    A common use case would be *ChRIS* *ds* plugins which operate
106    on individual input files.
107
108    `PathMapper` has four "constructors": `PathMapper.file_mapper`,
109    `PathMapper.dir_mapper_shallow`, and `PathMapper_dir_mapper_deep`
110    are for common uses cases whereas calling `PathMapper` directly
111    is for advanced usage. `PathMapper.file_mapper` is the most common
112    one: it yields input and output file pairs. If finer control over
113    the behavior is necessary, then use `PathMapper` with a custom
114    function given as the `filter`.
115
116    Examples
117    --------
118
119    Examples in this section are advanced tips and tricks.
120
121    Use [ThreadPoolExecutor](https://docs.python.org/3/library/concurrent.futures.html#threadpoolexecutor)
122    to parallelize subprocesses (akin to the usage of GNU
123    [parallel](https://www.gnu.org/software/parallel/)):
124
125    ```python
126    from pathlib import Path
127    import subprocess as sp
128    from concurrent.futures import ThreadPoolExecutor
129
130    # example: submit tasks one-by-one
131    with ThreadPoolExecutor(max_workers=4) as pool:
132        for input_file, output_path in PathMapper(input_dir, output_dir):
133            pool.submit(sp.run, ['external_command', input_file, output_path])
134
135    # example: call a function over all inputs
136    def do_something(input_file: Path, output_file: Path) -> None:
137        ...
138
139    with ThreadPoolExecutor(max_workers=4) as pool:
140        results = pool.map(lambda t: do_something(*t), PathMapper(input_dir, output_dir))
141
142    # iterate over results to raise any Exceptions which occurred in threads
143    for _ in results:
144        pass
145    ```
146
147    Hint: `` gets the number of CPUs available
148    to a containerized process (which can be limited by, for instance,
149    `docker run --cpuset-cpus 0-3`)
150
151    Add a progress bar with [tqdm](https://github.com/tqdm/tqdm):
152
153    ```python
154    from tqdm import tqdm
155
156    with tqdm(PathMapper(input_dir, output_dir)) as bar:
157        for input_file, output_path in bar:
158            do_something(input_file, output_path)
159    ```

340    @staticmethod
341    def _is_deep_dir(p: Path) -> bool:
342        """

344        """
345        if not p.is_dir():
346            return False
347        dirs = filter(lambda subpath: subpath.is_dir(), p.glob("*/"))
348        return next(dirs, None) is None

An iterator which discovers input files in a directory and maps them to output path names.

For example, an input file /share/incoming/a/b/c.txt will be mapped to /share/outgoing/a/b/c.txt.

A common use case would be ChRIS ds plugins which operate on individual input files.

PathMapper has four "constructors": PathMapper.file_mapper, PathMapper.dir_mapper_shallow, and PathMapper_dir_mapper_deep are for common uses cases whereas calling PathMapper directly is for advanced usage. PathMapper.file_mapper is the most common one: it yields input and output file pairs. If finer control over the behavior is necessary, then use PathMapper with a custom function given as the filter.

Examples

Examples in this section are advanced tips and tricks.

Use ThreadPoolExecutor to parallelize subprocesses (akin to the usage of GNU parallel):

from pathlib import Path
import subprocess as sp
from concurrent.futures import ThreadPoolExecutor

# example: submit tasks one-by-one
with ThreadPoolExecutor(max_workers=4) as pool:
    for input_file, output_path in PathMapper(input_dir, output_dir):
        pool.submit(sp.run, ['external_command', input_file, output_path])

# example: call a function over all inputs
def do_something(input_file: Path, output_file: Path) -> None:
    ...

with ThreadPoolExecutor(max_workers=4) as pool:
    results = pool.map(lambda t: do_something(*t), PathMapper(input_dir, output_dir))

# iterate over results to raise any Exceptions which occurred in threads
for _ in results:
    pass
